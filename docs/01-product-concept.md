# JDHub Mobile – Product Concept Document

## 1. Project Name

JDHub Mobile

## 2. Current Version Focus

The first version of JDHub Mobile will focus on a **Food Log / Meal Planner** module.

This first module is not intended to be a full nutrition tracker yet. It will not calculate calories, protein, carbohydrates, fat, or other nutrition values in the first version. The goal is to help users consistently log meals, plan meals, track food quantities, record weight, and build meal logging habits.

## 3. Long-Term Vision

JDHub Mobile will eventually become the mobile companion app for JDHub. It will start as a food log and meal planner, but the system should be designed in a modular way so it can grow into a pocket multi-tool app.

Future modules may include:

- Task tracking
- Project tracking
- Habit tracking
- Journal entries
- Fitness tracking
- Personal accountability
- Notifications and reminders
- JDHub web system sync
- IMS-related tools
- Personal dashboard

Because of this, the first version should not be designed as a standalone “Meal Tracker only” app. It should be designed as **JDHub Mobile**, with the Meal Tracker as the first module.

## 4. Main Purpose of Version 1

The purpose of version 1 is to help users:

- Log meals by date
- Plan future meals
- Track whether meals were planned, eaten, or skipped
- Track food weight using practical units such as grams and kilograms
- Save recent and favorite foods
- Add daily notes
- Log body weight by date
- Support normal schedules and night-shift schedules
- Receive meal logging reminders based on their usual sleep time
- Track streaks and consistency
- Export or back up data later

## 5. Core Principle

The app should be honest and practical.

Version 1 will not guess nutrition values. It will not use outside AI or inaccurate food estimation. Instead, it will focus on consistent food logging and weight-based tracking.

Nutrition features may be added later, but the required database fields can be planned ahead so the app can expand without needing a full redesign.

## 6. Target Users

The target users are people who want a simple and practical way to track their meals and food habits without complicated nutrition calculations.

The app should support users with different schedules, including:

- Normal daytime users
- Night-shift workers
- Work-from-home users
- Users who eat after midnight
- Users with irregular sleep schedules

## 7. Key Design Direction

JDHub Mobile should be **local-first** in the beginning. The user should be able to use the app without needing an account or internet connection.

Later, the app can connect to the main JDHub system for backup, syncing, and multi-device access.

## 8. Data Protection Direction

Since users may enter important personal data over time, the app should eventually support:

- Local data storage
- Export backup
- Import backup
- Possible Google Drive backup
- Future JDHub cloud sync

The app should be designed carefully so users do not lose data when updating, reinstalling, changing phones, or reformatting.
